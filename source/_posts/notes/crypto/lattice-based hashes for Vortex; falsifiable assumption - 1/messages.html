Alexandre Belling
I am certainly not the most legitimate person to answer this question, so please, anyone, do not hesitate to correct me. But as I understand it, an assumption is falsifiable if you can concretely make an experiment that shows everyone that the assumption has been broken if you manage to break it. For instance, you could convince me that you broke the collision resistance of a hash function by showing me to preimage that give the same result. On another hand, some EC pairing assumptions (like KoE or the AGM) are of the form "you can't obtain *this* result unless you did *that*", which are mostly used to assess "knowledge soundness" of some protocols. In this case, you can't really disprove it. As I understand, it is not really possible to estimate their security.

This is consistent with my understanding (not completely sure about the security estimation point though). So using the DLP as an example, we can roughly specify the security game to define the DLP as follows:

- Charlie sends a random group element G to adversary
- Charlie generates a random field element, x, and computes H = xG
- Charlie sends H to the adversary
- The adversary returns x'
- The adversary wins if x' equals x (This is easy to decide)

Which matches with the notes from the eprint:

A falsiable assumption is one that can be modeled as an interactive game where:

- There is an efficient challenger (runs in polynomial time) and an adversary
- The challenger must be able to efficiently decide if the adversary won the game

All efficient (polynomial time) adversaries should have a negligible chance of winning (or else the assumption is broken/falsified)

Alin Tomescu
In reply to this message
The most insightful paper I’ve read on this is:

[Naor03] On Cryptographic Assumptions and Challenges; by Naor, Moni; in Advances in Cryptology - CRYPTO 2003; 2003 (https://link.springer.com/chapter/10.1007/978-3-540-45146-4_6)

But, real quickly, the easiest (but not fully-nuanced) way I think about them is:

With a falsifiable assumption, one can easily create a random _challenge_ for people to break the assumption and post this on a website. Then, anyone can post a solution claiming to have broken the assumption. Importantly, everyone will be able to verify the solution against the challenge.

For example, consider the discrete log assumption: I can post a random challenge (g, g^a) and wait for the solution ‘a’. Everyone can verify that this is the correct ‘a’.

A **non**-falsifiable assumption simply does not allow for something like this. [Naor03] formalizes this well, I think, if you want more detail.

But an example here could make this point clearer: The knowledge of exponent (KoE) assumption says that given (g, g^x), an adversary who outputs (g^y, g^{xy}) must “know” y. This is modeled as \forall adversaries A, \exists an extractor E that would output y. To falsify this assumption, one would try and prove that \exists an adversary A, \forall extractors E, the extractor E cannot output the correct y. In other words, one must prove that \exists an adversary A for which there is no extractor. So there’s no challenge posted on a website that one can break. One must simply prove the assumption false.

A beautiful example of how a **non**falsifiable assumption can actually be falsified using the strategy above can be found in:

[BP04] The Knowledge-of-Exponent Assumptions and 3-Round Zero-Knowledge Protocols; by Bellare, Mihir and Palacio, Adriana; in Advances in Cryptology -- CRYPTO 2004; 2004 (https://eprint.iacr.org/2004/008.pdf)

Here, the authors show that if the knowledge-of-exponent-assumption-2 (KEA2) holds, then discrete log is easy.
